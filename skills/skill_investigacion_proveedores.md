# Skill: Investigación y Desarrollo de Proveedores

## Cuándo usar esta skill

Cuando el presupuesto requiere materiales, servicios o subcontratas sin proveedor claro, sin precio fiable o con necesidad de alternativas.

También se usa para completar datos de proveedores existentes antes de pedir precios.

## Datos mínimos necesarios

- Material o servicio a investigar.
- Cantidad estimada y unidad.
- Especificación técnica mínima.
- Ubicación de entrega o zona de obra.
- Urgencia o fecha límite para recibir precio.

## Datos recomendados

- Producto final que se está presupuestando.
- Calidad, acabado, certificado o norma requerida.
- Preferencia de proveedor local, nacional o online.
- Precio objetivo si existe.
- Proveedor habitual si ya se conoce.

## Fuentes de investigación

- Fichas existentes en `/proveedores/fichas/`.
- Búsquedas previas en `/proveedores/busquedas/`.
- Webs de proveedores y fabricantes.
- Directorios locales o sectoriales.
- Contactos habituales de HAM.
- Resultados web cuando sea necesario buscar nuevas alternativas.

## Proceso

1. Definir la necesidad técnica en una línea clara.
2. Consultar si existe proveedor validado en `/proveedores/fichas/`.
3. Si no existe, buscar proveedores candidatos.
4. Registrar la búsqueda en `/proveedores/busquedas/` si aporta información reutilizable.
5. Crear o actualizar una ficha de proveedor.
6. Aplicar `proveedores/checklist-cualificacion.md`.
7. Si el proveedor es válido, preparar email o WhatsApp de solicitud de precio.
8. Al recibir respuesta, registrar precio, plazo, IVA, transporte y validez.
9. Actualizar el coste del presupuesto o dejar la partida como pendiente de confirmación.

## Criterios de cualificación

- Sirve la zona de entrega o puede enviar a Menorca.
- Puede suministrar la especificación requerida.
- Responde con precio claro y condiciones.
- Indica plazo y disponibilidad.
- Indica IVA y transporte.
- Puede ser reutilizado en futuros presupuestos.

## Formato de salida recomendado

El agente debe entregar:

1. **Resumen de necesidad:** material/servicio, cantidad, especificación y urgencia.
2. **Proveedores candidatos:** nombre, contacto, web, motivo por el que encaja.
3. **Acción recomendada:** contactar, pedir precio, crear ficha o descartar.
4. **Mensaje listo para enviar:** email o WhatsApp si se solicita contacto.

## Checklist final

- [ ] Necesidad técnica definida.
- [ ] Se consultaron proveedores existentes.
- [ ] Se buscaron alternativas si no había proveedor claro.
- [ ] Datos de contacto registrados.
- [ ] Proveedor cualificado o marcado como candidato.
- [ ] Se preparó mensaje de contacto si procede.
- [ ] Se indicó qué datos faltan para cerrar el coste.

## Errores frecuentes a evitar

- Guardar un proveedor sin categoría.
- No registrar fecha del precio recibido.
- No distinguir candidato de proveedor validado.
- Usar precio sin saber si incluye transporte o IVA.
- No buscar alternativas cuando el proveedor es crítico.
